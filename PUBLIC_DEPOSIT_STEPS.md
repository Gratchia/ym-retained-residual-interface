# Public Repository / DOI Deposit Steps

## Important

This package is ready for public deposit, but the assistant cannot directly create a public repository or DOI from this environment.

## Recommended route

1. Create a new GitHub repository, for example:
   `ym-retained-residual-interface`

2. Upload the contents of this package to the repository.

3. Create a GitHub release, for example:
   `v0.2`

4. Link GitHub to Zenodo and enable the repository. Zenodo's GitHub integration can archive releases and assign a DOI.

5. After Zenodo creates the DOI, provide the DOI/URL back in chat.

6. The manuscript should then be updated in the Code and supplementary material paragraph by replacing:
   `A public repository link or archival DOI should be inserted here before journal submission.`
   with the actual repository URL/DOI.

## Do not upload the source PDFs

Do not include the Batrouni 1982 or Borisenko--Voloshin--Faber PDFs in the public repository unless rights explicitly permit redistribution.
